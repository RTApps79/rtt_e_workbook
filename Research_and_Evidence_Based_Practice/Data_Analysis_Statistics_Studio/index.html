Data Analysis Using Statistics Studio

Interactive Lesson

Page 1: No SPSS Required

This module traditionally requires access to IBM SPSS through a university VDI or portal login. Here, you'll run the exact same five analyses using RTApps Statistics Studio — a free, in-browser statistics tool built for this course. No installation, no license, no VDI wait times. Your data never leaves your browser.

The statistical reasoning is identical either way — you're still forming hypotheses, choosing the right test, and interpreting output. Only the click-path changes.

Page 2: Translating SPSS Concepts to Statistics Studio

If you've seen SPSS terminology before, here's the direct translation:

SPSS termStatistics Studio equivalent
Data ViewThe dataset preview table after you load a CSV
Variable ViewNot needed — Statistics Studio detects numeric vs. categorical columns automatically
Output ViewerThe results cards that appear below each analysis panel
Sig. (2-tailed)p — shown directly in each results card
Analyze menuThe numbered sidebar (Data, Descriptive, Frequencies, Group Tests, Relationships, Categorical…)

Page 3: Loading the Dataset

Open Statistics Studio and go to 1. Data. Upload RT_PatientOutcomes_Sample.csv directly — no import wizard, no manual variable type declarations.

One real prep step remains: for the Chi-Square analysis later, you'll need Satisfaction_Score (1–5) collapsed into a two-level High/Low category. Do this in a spreadsheet before uploading — add a column with a formula like =IF(Satisfaction_Score>=4,"High","Low"), save, then upload. This is a completely standard data-prep step in any real research workflow, not something unique to SPSS.

Page 4: Analysis 1 — Descriptive Statistics

Go to 2. Descriptive. Select Age, run it, record the mean and SD. Repeat for Anxiety_Pre, Anxiety_Post, Setup_Error_mm, and Fractions_Completed.

For Treatment_Site and Sex, go to 3. Frequencies instead — these are categorical, so you want counts and percentages per category, not a mean.

Every published study opens with this step: describe your sample before testing anything.

Page 5: Analysis 2 — Paired-Samples t-Test (Tests H1)

Go to 4. Group Tests → Paired-samples t test. Set Anxiety_Pre as the first variable and Anxiety_Post as the second. Run it.

Same 50 patients, measured twice — that's what makes this paired rather than independent. Record the mean difference, t, df, p, and the 95% confidence interval directly from the results card.

Quick Check!

Why is a paired t-test the correct choice for comparing Anxiety_Pre to Anxiety_Post?

B. Pairing accounts for each patient's own baseline, which is only valid when the same subjects are measured twice.

Page 6: Analysis 3 — One-Way ANOVA (Tests H2)

Still in 4. Group Tests, find the One-way ANOVA card. Set Treatment_Site as the group variable and Setup_Error_mm as the numeric outcome. Run it.

One real difference from SPSS: Statistics Studio's post-hoc pairwise comparisons use a Bonferroni correction rather than Tukey HSD. Both control for the fact that testing multiple pairs inflates your false-positive risk — Bonferroni is simply a more conservative, more transparent method to compute by hand or in open-source code. Your conclusions (which pairs differ) will typically match; the exact p-values will not match an SPSS Tukey table digit-for-digit.

Record the F statistic, both degrees of freedom, p, and η² from the ANOVA table, then the three pairwise comparisons from the post-hoc table.

Page 7: Analysis 4 — Chi-Square Test (Tests H3)

Go to 6. Categorical. Set Education_Received as the row variable and your recoded Sat_HighLow column as the column variable (the one you created in Page 3). Run it.

Check the observed and expected counts table first — if any expected cell count is below 5, that's a real limitation to report, not something to ignore.

Quick Check!

If an expected cell count in your chi-square table is below 5, what should you do?

A. Expected counts below 5 violate a key chi-square assumption and should be explicitly noted as a limitation.

Page 8: Analysis 5 — Pearson Correlation (Tests H4)

Go to 5. Relationships → Pearson correlation. Set Therapist_Experience_Yrs as X and Setup_Error_mm as Y. Run it — Statistics Studio also draws the scatterplot automatically.

Record r, p, direction, and strength (Statistics Studio labels the strength descriptor for you using Cohen's 1988 guidelines — no need to look it up separately).

Page 9: Synthesis

Across your four hypothesis tests (H1–H4), you now have a decision for each: reject or fail to reject the null. Pull those four decisions together and think about which single finding would be most actionable for a radiation therapy department to act on immediately.

Applying this to your own work: which of these five analysis types — descriptive, paired t, ANOVA, chi-square, or Pearson r — is most likely to fit a research question you'd want to investigate yourself? That choice comes directly from the variables you'd be measuring and their NOIR level (see the Variables & Measurement lesson).
