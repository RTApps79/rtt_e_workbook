Console Operations & Interlocks

Interactive Lesson

Page 1: Why Interlocks Exist

An interlock is a safety condition the machine continuously checks. If a condition isn't met, the machine won't start the beam — or if it's already running, it stops immediately. Interlocks aren't obstacles to work around; they're the reason a LINAC is safe to operate at all.

Interlocks fall into three categories, and knowing which category you're dealing with tells you what to do next: Minor, Major, and Dosimetry.

Page 2: Minor Interlocks

Minor interlocks involve conditions that can easily be corrected — the machine flags something simple and lets you fix it on the spot.

CodeMeaning
DOORThe treatment room door is open
ACCThe accessory mounted on the treatment unit doesn't match what was selected at the console
KEYAn issue with the keyswitch/key-based access control
MLCAn MLC-related positioning or communication issue
PNDTAn issue detected via the hand pendant

To clear a DOOR interlock, for example, you simply close the door. That's the defining feature of a minor interlock — the fix is usually obvious once you know what the code means.

Page 3: Clearing Minor Interlocks

The exact clearing procedure depends on when the interlock is detected:

At end of calibration/check cycle: the console displays an active interlock list with the message "MINOR INTERLOCK — # MU REMAINING" or "MINOR INTERLOCK, PRESS ENTER TO CONTINUE." If "PRESS ENTER TO CONTINUE" appears, pressing Enter clears it. Otherwise, you correct the actual cause (e.g., close the door) before it clears.

During beam-on: the beam terminates immediately and all motion stops. The same message pattern applies — correct the cause, or if it can't be cleared, record the MU and time remaining and remove the patient from the room if appropriate.

Quick Check!

A DOOR interlock trips mid-treatment. What's the immediate machine response?

A. Interlocks detected during beam-on cause immediate termination — the machine doesn't wait to finish the current delivery.

Page 4: Major Interlocks

Major interlocks alert you to conditions that could actually damage the accelerator if not corrected — these are treated with more caution than minor interlocks.

CodeMeaning
CTRLA control system fault — requires running diagnostics to clear
KFIL / MFILKlystron or magnetron filament-related fault
PUMPA pump system fault (e.g., vacuum pump)
VAC1A vacuum system fault
FLOWA cooling flow system fault

Page 5: Clearing Major Interlocks

The procedure is more involved than a minor interlock:

  1. The console displays the SELECT MAJOR MODE screen and an active interlock list, with a message to turn the power key to STANDBY.
  2. Turn the POWER keyswitch to STANDBY.
  3. Record in the daily log or service log: date, time, interlock type, system state (including all setup/position parameters), and any other observations. If it happened during beam-on, record MU delivered and elapsed time too.
  4. Check the cause. Correct it if possible, or notify the physicist/repair technician. Remove the patient from the room if appropriate.
  5. For a CTRL interlock specifically, you must run diagnostics (F2 DIAG) while still in STANDBY before it will clear.
  6. Turn the key back to ON. If the interlock reappears after clearing, do not continue — this is not something to retry repeatedly.

The documentation step isn't optional paperwork — a service history that shows an interlock recurring over time is exactly what helps a physicist or repair technician catch a developing problem before it becomes a bigger failure.

Page 6: Dosimetry Interlocks

The beam passes through the dual ion chambers, which continuously check flatness, symmetry, and other dosimetric qualities. Dosimetry interlocks are triggered when these readings fall outside tolerance.

Because momentary beam variations are common and usually transient, a second attempt is often successful. However, clearing a dosimetry interlock requires entering a password to continue — a deliberate extra step, since these interlocks relate directly to whether the delivered dose was actually correct.

CodeMeaning
DOS1Primary dosimetry channel reached the set MU value — this is actually the normal beam termination signal, not a fault
SYM1 / SYM2Beam symmetry outside tolerance
ION1Primary ion chamber reading issue

Quick Check!

What must you do to clear a dosimetry interlock and continue treatment?

B. Dosimetry interlocks require a password to continue, since they relate directly to whether the delivered dose was actually correct — an intentional extra safety step.

Page 7: Reading an Interlock Code

Real interlock codes are short, specific, and traceable to a particular subsystem. A few more examples worth recognizing:

CodeCategoryMeaning
AIRMinorFacility air pressure below spec — can indirectly trigger other interlocks like TARG or TDRV
BMAGMinorBending magnet power supply or current outside tolerance for the selected energy mode
CARRMinorCarrousel (energy/mode selector) not in the correct position, or a communication failure with its controller
ENSWMinorThe accelerator's energy switch isn't in the correct position for the selected mode
CNFMinorConfiguration data (field sizes, position readouts, motion limits) wasn't properly sent to the control computer

Notice the pattern: each code maps to one specific subsystem, and the "possible reasons" behind it are usually a short, specific list — not a vague catch-all.

Page 8: Beyond Interlocks — Other Safety Features

Interlocks work alongside other safety systems already covered elsewhere: warning lights and signs, emergency-off controls placed at multiple points in the room, and motion-enable bars that require continuous physical input from the therapist to allow couch/gantry motion — releasing the bar stops movement immediately. Together, these form layered protection: procedural (interlocks), physical (emergency-offs, motion-enable), and environmental (shielding, warning systems).

Page 9: Wrap-Up

You've covered the three interlock categories (Minor, Major, Dosimetry), how each is detected and cleared, specific real interlock codes across several LINAC subsystems, and how interlocks fit into the broader layered safety system.

None of this replaces hands-on training on your specific machine and vendor — interlock codes and exact clearing procedures vary between manufacturers and software versions — but the underlying logic (what category, what it protects, how you clear it) transfers across machines.
